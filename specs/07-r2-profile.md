# OLOS Cloudflare R2 Profile

Status: `draft-v0.1.2`

## 1. Abstract

The OLOS R2 Profile defines how to implement OLOS Object Binding using Cloudflare R2.

This profile now uses **one R2 bucket and no copy/promotion step** as the default model.

This is a vendor profile. It is not OLOS Core.

## 2. Deployment model

Recommended R2 deployment:

```text
R2 bucket: olos-media
  receives pre-signed PUT uploads
  serves committed media through a Cloudflare custom domain

Cloudflare Queue or equivalent
  receives R2 object-created notifications, where used

Coordinator Worker
  issues upload slots
  signs pre-signed PUT URLs
  observes object-created events or verifies client completion hints via HeadObject
  commits slots
  advances cursor and CommittedWindow

Gateway Worker
  renders LL-HLS playlists from cursor and CommittedWindow
```

The same object key is used for upload and delivery.

## 3. Bucket and key layout

Single bucket:

```text
Bucket: olos-media
Prefix: media/{tenant}/{session}/...
```

Canonical object keys:

```text
media/{tenant}/{session}/e{epoch}/{rendition}/init-{slot_nonce}.mp4
media/{tenant}/{session}/e{epoch}/{rendition}/s{msn}/p{part}-{slot_nonce}.m4s
media/{tenant}/{session}/e{epoch}/{rendition}/s{msn}/segment-{slot_nonce}.m4s
```

The per-slot nonce SHOULD be generated by the coordinator and SHOULD make future object URLs non-obvious.

## 4. Public access model

This profile uses manifest-gated publication.

```text
object exists in R2        != official stream state
object committed into cursor/CommittedWindow == official stream state
```

A public single-bucket deployment MAY allow direct reads of uploaded objects before commit if the URL is known. This is an accepted trade-off of this profile. Deployments that require uncommitted objects to be unreadable MUST add read gating or use private upload/public promotion.

Production deployments SHOULD use an R2 custom domain for public media delivery so Cloudflare cache, WAF rules and access controls can be applied. Public `r2.dev` access SHOULD be disabled in production.

## 5. Pre-signed PUT

Each upload slot SHOULD receive one pre-signed PUT URL.

Required properties:

```text
exact object key
short expiry
method PUT
Content-Type: video/mp4
If-None-Match: *
slot metadata where possible
```

Example request:

```http
PUT https://<account>.r2.cloudflarestorage.com/olos-media/media/tenant/sess/e1/v1080/s3812/p3-slot_01JZ.m4s?...signature...
Content-Type: video/mp4
If-None-Match: *
x-olos-slot-id: slot_01JZ
```

The coordinator MUST treat the pre-signed URL as a bearer capability and SHOULD use short expiries for live parts.

## 6. Conditional upload

The R2 profile SHOULD use conditional `PutObject` with:

```http
If-None-Match: *
```

The coordinator SHOULD include this header in the signed request where practical.

If conditional upload cannot be enforced by the provider/signing path, the coordinator MUST detect overwrite or duplicate conflicts before commit and MUST NOT announce overwritten objects.

## 7. Object-created notifications

R2 event notifications SHOULD be configured for the media prefix.

Event filters:

```text
prefix: media/
suffix: .m4s or .mp4
```

The event consumer MUST:

```text
normalise R2 event into OLOS ObjectCreatedEvent
ignore unknown prefixes
lookup matching upload slot by object key
check object size and etag
mark slot upload_observed
commit if policy allows
handle duplicate events idempotently
```

An R2 object-created event is not by itself publication authority. It only proves that an object candidate exists.

## 8. Client completion hints

Publishers MAY call:

```http
POST /olos/v1/sessions/{session_id}/upload-slots/{slot_id}/complete
```

The hint MAY include:

```json
{
  "etag": "\"...\"",
  "size": 312500
}
```

The coordinator MUST treat this as a hint. It MUST verify by `HeadObject` or provider event before commit.

## 9. Commit flow

Recommended flow:

```text
receive object event or completion hint
HeadObject exact object key if needed
validate key, slot, size, content type, etag and session state
commit upload slot
advance cursor and CommittedWindow
manifest gateway emits object URL
```

No media byte parsing is required in No-Scan Mode.

## 10. Public delivery

Public media SHOULD be served through an R2 custom domain.

Recommended media headers:

```http
Content-Type: video/mp4
X-Content-Type-Options: nosniff
Cache-Control: public, max-age=31536000, immutable
Access-Control-Allow-Credentials: false
Set-Cookie: never
```

Configure Cloudflare rules to:

```text
deny unknown extensions
deny top-level document navigation to media objects
deny HTML Accept requests for media objects
avoid caching 404s for live future paths
avoid overwrites entirely
```

## 11. Cache caveats

The coordinator SHOULD verify object existence using R2's S3 API or Worker binding path, not through the public cached custom domain.

The manifest gateway SHOULD NOT emit future deterministic object URLs. LL-HLS preload hints SHOULD be disabled by default for this profile unless future-object requests are routed through a blocking non-cached path.

## 12. Cursor and state storage

The R2 profile does not require a specific cursor or CommittedWindow store.

Possible implementations:

```text
Durable Object
Workers KV plus Durable Object lock
D1/Postgres
Redis
application database
```

The cursor/CommittedWindow store MUST provide safe monotonic updates.

## 13. Worker split

Recommended separation:

```text
Coordinator Worker
  authenticated publisher/control API
  slot issuance
  commit processing
  event handling
  quotas
  kill switch

Gateway Worker
  public or viewer-authenticated playback API
  master.m3u8
  media.m3u8
  blocking reload
```

Media bytes SHOULD NOT flow through Workers in the normal path.

## 15. R2-specific security notes

```text
pre-signed PUT URLs are bearer capabilities
object keys must be exact and coordinator-generated
If-None-Match: * should be enforced
object existence must be verified before playlist emission
public direct-read before commit is possible in this profile
custom domain rules should prevent media objects being treated as documents
404 caching must be safe for live prefixes
```

## 16. Implementation checklist

```text
[ ] one R2 bucket created
[ ] public custom domain configured
[ ] production r2.dev access disabled
[ ] pre-signed PUT uses exact object key
[ ] Content-Type is bound to video/mp4
[ ] If-None-Match: * is required where practical
[ ] R2 events or HeadObject verification implemented
[ ] event consumer is idempotent
[ ] coordinator never playlists unobserved objects
[ ] object key includes per-slot nonce
[ ] preload hints disabled by default
[ ] 404 caching safe for media prefix
[ ] media origin headers configured
[ ] document navigation to media objects blocked
[ ] quotas and kill switch implemented
```
