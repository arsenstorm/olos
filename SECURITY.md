# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in OLOS, do not create a
public issue, discussion, or pull request.

Use GitHub's private vulnerability reporting flow for this repository:

https://github.com/arsenstorm/olos/security

Useful reports include:

- a clear description of the vulnerability and its impact
- steps to reproduce or a proof of concept
- the affected OLOS package surface, version, or commit SHA
- any relevant logs, screenshots, requests, or generated manifests with secrets
  and personal data redacted
- the best way to contact you for follow-up questions

## Disclosure

Please do not publicly disclose the issue until it has been investigated and a
fix or mitigation has been coordinated. Responsible disclosures will be credited
when appropriate and requested.
